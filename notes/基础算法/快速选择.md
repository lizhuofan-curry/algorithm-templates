``` c++
#include <bits/stdc++.h>
using namespace std;
const int N=1e5+10;
int q[N];
int quick_select(int l,int r,int m)
{
    if(l>=r) return q[l];
    int i=l-1,j=r+1,x=q[l+r>>1];
    while(i<j)
    {
        do i++;while(q[i]<x);
        do j--;while(q[j]>x);
        if(i<j) swap(q[i],q[j]);
    }
    int num=j-l+1;
    if(m<num)return quick_select(l,j,m);
    else return quick_select(j+1,r,m-num);
}
int main()
{
    int n,k; cin>>n>>k;
    for(int i=0;i<n;i++) cin>>q[i];
    cout<<quick_select(0,n-1,k-1);
    return 0;
}
```
# 快速选择中的易错点
快速选择和快速排序的差别其实不大，易错点相对快速排序来说就一个地方：  
在确定区间后，我们要算出左边区间数字的数量num=j-l+1   
然后比较num和m的大小关系  
如果是小于的话 就只需更改递归区间   
如果是大于的话 就要在更改递归区间的同时修改m的值（因为左区间有num个数小于要查找的了，而我现在到了右区间中，所以我的m=m-num）
## 更新一处易错点
在快速选择调用函数时，正常来说是你要查找第几大的数就将几传入函数中
但是我刚刚重写了一遍发现了一个问题：  
如果我在最后判断时用的是m<num,那么我在调用函数时得将我要查找的数减1,
因为此时我将原数看成了以0开头的
